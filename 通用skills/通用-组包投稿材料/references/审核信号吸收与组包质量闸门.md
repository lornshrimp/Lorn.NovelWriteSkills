# 审核信号吸收与组包质量闸门

## 这份规则解决什么问题

投稿组包最容易犯的错，不是“缺文件”，而是**上游明明没过线，却被包装成像能投**。

因此，组包前必须先把“签约评估脑 + 审阅脑”吸进来，再决定：

1. 停止正式组包；
2. 只做可投包预备版；
3. 允许正式组包。

## 必须优先吸收的结论源

只要材料存在，组包时默认优先吸收：

- `通用-平台签约评估框架`
- `通用-审阅章节正文`
- `通用-审阅人物传记`
- `通用-审阅技术设定`
- `通用-审阅分卷大纲`
- `通用-审阅总大纲`

## 三档质量闸门

### A. 停止正式组包

命中以下任一类，原则上先回炉，不做正式投稿包：

- **章节层**：存在功能断裂、事实重大冲突、关键线索不公平、前门无有效回报。
- **人物层**：主角或重要对手动机 / 代价塌陷，能力万能，首秀失败，读者联结极弱。
- **设定层**：规则互相打架、没有边界、没有痕迹、没有现实锚点。
- **分卷层**：卖点过少 / 过晚 / 过弱，本卷只有推进没有回报，首卷承诺失效。
- **总纲层**：主卖点两卷以上失踪，主支线职责混乱，“首卷亮、中段断供”。

### B. 可投包预备版

适用于：

- 材料还不完整；
- 审阅层没有致命问题，但存在明显未修完的中高风险；
- 书名、简介、样章、大纲还没完全统一；
- 长线和执行准备还不足以支撑“正式提交”。

这时必须显眼标注：**可投包预备版，不是正式提交版。**

### C. 允许正式组包

至少同时满足：

- 题材与平台可快速归类；
- 前门样章能抓住编辑继续看；
- 主角、机制、代价与卖点已同框成立；
- 首卷承诺与长线供血清楚；
- 更新与投稿材料具备基本可信度。

## 五类审阅信号该如何翻译成组包动作

### 1）章节正文信号 → 样章裁判

- `P0 / P1` 风险未修掉：停止正式组包或降级为预备版。
- 章首抓力、中段回报、章末接棒对象不足：优先改样章，不先改大纲包装。
- 证据公平或规则边界不足：样章不得直接交付。

### 2）人物传记信号 → 卖点与简介裁判

- 主角一句话定义不成立：简介重写前置。
- 动机—代价闭环不足：样章与大纲都要改，不可只修简介。
- 大纲承诺的人物功能与传记不一致：整包降级。

### 3）技术设定信号 → 机制与说明裁判

- 规则互相打架 / 边界缺失：停止正式组包。
- 只有解释、没有动作链：样章和大纲都需回炉。
- 术语、编号、时间规定口径不统一：视为材料内部失分点。

### 4）分卷大纲信号 → 首卷承诺与中段续航裁判

- 前 5 章追读链不足：样章与大纲必须一起修。
- 卖点保真不足：简介与大纲不可继续夸大承诺。
- 单元案不能供血主线：大纲不可包装成“长线很稳”。

### 5）总大纲信号 → 长线潜力裁判

- 主线对象与主支线职责混乱：停止正式组包。
- 主卖点未进入主线系统：大纲必须重写。
- 首卷承诺到终局不一致：只能做预备版或先回炉。

## 组包自检至少要回看什么

1. 书名、简介、样章、大纲是不是同一套承诺。
2. 前 `300 / 500 / 1000 / 3000 / 6000` 字是不是都有人可继续看的理由。
3. 主角、机制、代价、回报是不是同时出现，而不是各说各话。
4. 首卷承诺、中段供血、长线推进是不是能互相印证。
5. 当前包最强的一环和最弱的一环分别是什么。

## 组包时最容易自欺的三种情况

- 简介很强，正文很弱。
- 大纲很大，样章很空。
- 设定很亮，人物和回报发虚。

只要命中其中一种，就不能把稿子包装成“风险已清”的正式版。
