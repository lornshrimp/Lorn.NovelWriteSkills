# 黄金三章与首轮回报设计

## 起点默认口径的黄金三章

编辑快审时，优先看的是：

- 前 300 字有没有明确冲突
- 前 500 字有没有继续看下去的理由
- 前 1000 字主角与异常机制是否已有效入场
- 前 3000 字是否完成第一次小闭环 / 小爽点

## 开篇风格路由（通用四选一）

不同题材可以换壳，但开篇抓人的底层路数高度共通。默认优先从以下四路里选一条主路，不要四条平均发力：

- **现实压迫开场**：先给主角一个当前就疼的现实处境，再让异常 / 冲突顶进来。
- **证据异常开场**：先抛出一个能被看到、听到、摸到、记录到的反常信号，再让人物被迫处理。
- **对话冲突开场**：用一句带位差、带风险、带潜台词的对话把关系和危机同时立起来。
- **行动追索开场**：主角已经在跑、找、躲、追、赶、试、查，让读者先跟上动作，再逐步补信息。

选择原则：

- 想强化代入，优先“现实压迫开场”或“行动追索开场”。
- 想强化悬念，优先“证据异常开场”或“对话冲突开场”。
- 开篇只保一条主路，其他效果作为辅路叠加，避免首屏像功能清单。

## 章首三句钩子最低要求

无论采用哪一路，章首前三句最好尽快完成以下三件事中的至少两件：

- 抛出一个**具体异常 / 具体风险**，而不是抽象气氛。
- 让读者知道主角**此刻会失去什么**。
- 让主角出现一个**立刻可见的动作**，而不是只有判断和回忆。

如果前三句里既没有异常、也没有代价、也没有动作，通常说明章首还停留在热身。

## 第一章的最低任务

- 先给主角一个当前就会疼的现实处境
- 在处境里嵌入一个反常信号
- 让主角必须做出一个动作，而不是只观察
- 章末留下更危险的问题，而不是只留气氛

## 第二章的最低任务

- 把第一章抛出的异常钉到规则、证据或代价上
- 让主角第一次试出规则边界
- 给出短期目标：活下去、保住谁、拿到什么、查清哪件事
- 继续放大失手风险

## 第三章的最低任务

- 兑现一次首轮回报：反击、验证、误导被证伪、拿到关键证据、逼退一个具体威胁
- 让读者看到“这条线值得继续追”
- 章末必须引出更大的新变量

## 代入感四支柱（通用抓手）

若希望正文不是“会讲故事”，而是“让人跟着进场”，优先检查四个支柱是否到位：

1. **现实痛点**：主角眼下到底被什么现实问题掐着。
2. **可验证异常**：危险或机会必须有可感知的外显载体。
3. **立即动作**：主角必须做点什么，哪怕只是试探、回拨、跟上、打开、核对。
4. **可感代价**：失手后会疼在哪里，最好能被读者直接想象。

四支柱不必平均铺开，但前一两千字里最好都能看到影子。

## 首轮回报的合格形式

以下至少命中其一：

- 主角第一次有效破局
- 关键规则被试出来
- 一条误导被推翻
- 一件关键物证落袋
- 一段关系发生不可逆变化
- 一次小规模打脸或认知反转成立

## 掌控感交付（首轮追读补丁）

起点、平台连载和多数长篇题材都不适合让主角前几章一直纯挨打。哪怕整体局势仍差，前 1–3 章也最好交付一次“主角更接近掌控局面”的感觉，例如：

- 主角看出别人没看出的破绽
- 主角抢先一步拿到、验证或锁定关键要素
- 主角第一次把规则、流程、空间或关系用成自己的优势
- 主角虽然没赢，但成功把“只能被动承受”改成“可以主动试探”

这种掌控感不等于无敌感，而是给读者一个明确承诺：主角不是只会被剧情推着走。

## 常见失败信号

- 300 字后仍在解释背景
- 主角只想，不做
- 机制很晚才显形
- 三章过去还没有任何明确回报
- 章末只留“他有种不祥预感”之类空钩子

## 默认修正优先级

1. 先删掉无效背景热身
2. 再把异常 / 冲突前置
3. 再让主角更早动作
4. 最后把首轮回报钉成具体事件
