# 九部件预埋与进度节奏微表

适用于章纲有了、素材也有了，但控制卡仍然写不出主推进线、中段回报或下一章接口的情况。

## 一个原则

控制卡写不稳，通常不是想法不够，而是本章尚未放回故事整体位置里看。

所以在生成控制卡前，先用九部件确认本章所在位置，再压出微型进度节奏表。

## 先确认本章位于哪一段

### 四部分定位

- 布局章：负责交代现状、缺口、重大危机前的稳定秩序
- 反应章：负责震后余波、撤退、试探、整理、误判
- 进攻章：负责主动出击、改策略、拿主动权
- 解决章：负责最后摊牌、兑现、代价与收束

如果本章既想做反应，又想做终局解决，通常会发散。

## 再确认本章靠近哪个里程碑

- 是在铺引发性事件？
- 是在走向第一情节点？
- 是在把主人公送向中间点？
- 是在中间点后推动进攻？
- 是在为第二情节点做最后蓄压？
- 还是已经进入解决段？

## 控制卡生成前的微表

每章先压 3–5 个节拍即可：

1. 开场压力：这一章一开始最紧的事是什么？
2. 试探 / 推进：主角现在做的第一步是什么？
3. 新阻碍 / 新信息：中段必须给什么回报？
4. 代价 / 改判：本章最值钱的变化是什么？
5. 章末驱动：下一章为什么必须接着写？

## 把九部件翻译成控制卡语言

### 若本章属于“反应段”

控制卡要重点写：

- 主角在躲什么 / 想搞清什么
- 为什么现在还不能直接翻盘
- 本章中段给到哪种理解或误判回报

### 若本章属于“进攻段”

控制卡要重点写：

- 主角第一次真正主动做什么
- 这次行动如何体现人物变化
- 本章末尾留下哪种更大的阻力

### 若本章属于“解决段”

控制卡要重点写：

- 最后一批新信息是否已经到齐
- 主角如何成为解决问题的催化剂
- 收束与余波怎样分层

## 控制卡失败警报

出现以下情况，先不要急着写正文：

- 本章核心改变说不成一句话
- 中段回报只能写成“继续推进”
- 章末钩子只是情绪感叹，没有新驱动
- 不知道本章和上一章、下一章的结构关系

## 最低输出格式

- 本章结构位置：四部分中的哪一段
- 本章关联里程碑：承上 / 启下哪个转折点
- 3–5 个节拍微表
- 本章中段回报
- 本章章末驱动
- 本章与主卖点的关系：强化 / 兑现 / 转化 / 保温 / 铺垫

## 注意

- 微表不是摘要，而是控制卡的骨架。
- 真正好用的控制卡，必须让你知道“下一场写什么”，而不是只知道“这一章大概讲什么”。
