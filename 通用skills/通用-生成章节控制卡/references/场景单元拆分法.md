# 场景单元拆分法

## 一、不要按“段落”拆，要按“压力变化”拆

一个有效场景单元，至少包含：

- 目标
- 阻力
- 小结果

只换了地点、没换压力，不一定值得拆成新单元。

## 二、常用 4 单元结构

### 单元 1：晚进入场

开在压力已经在场的位置，而不是暖机。

### 单元 2：试探 / 误判

角色以为自己知道问题，但其实还在错路上。

### 单元 3：中段回报

必须让读者“拿到东西”。

### 单元 4：代价 / 钩子

局部完成，但更大的债务立起来。

## 三、五单元扩展结构

当一章较长或任务较重时，可扩成：

1. 入场压力
2. 局部推进
3. 中段回报
4. 误判修正 / 代价升级
5. 章末钩子

## 四、拆分失败信号

- 每个单元只是“继续聊 / 继续走 / 继续想”
- 说不出第三单元的中段回报
- 最后一个单元只有感慨，没有动作 / 信息 / 危险

## 五、先钉问题，再钉张力，再看是否解决

拆控制卡时，先问本章真正的问题是什么：

- 谁要什么？
- 谁在拦？
- 现在最危险的失去是什么？

再看张力怎样被拉长：

- 是误判带来的拖延
- 是关系上的不信任
- 是程序上的卡口
- 还是行动中不断上涨的代价

最后再决定本章是否解决：

- 彻底解决
- 局部解决但引出更贵问题
- 根本没解决，只是把真正问题暴露出来

如果连这三步都说不清，控制卡往往只是梗概，不是施工卡。

## 六、用“焦点”来判断单元顺序

一个章节单元，可以粗看成若干个焦点的推进：

1. 发现问题
2. 发展问题
3. 暂时解决或把问题推到更危险的位置

拆场时优先看“焦点是不是变了”，而不是“人是不是挪地方了”。

### 常见误拆

- 地点变了，但还是在说同一个东西
- 对话换了人，但压力没有升级
- 单元很多，但每个焦点都还在原地打转
控制卡拆到这里，至少应能一眼看出哪一段真的在把局面往前推。
