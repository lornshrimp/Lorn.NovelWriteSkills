# 通用-输出WebNovel版

这是平台共性本体，负责承接“WebNovel 版输出”的跨题材共性规则。

## 常见触发词 / 用户说法速查

- 输出WebNovel版
- 改成 WebNovel 能发的版本
- 做 WebNovel 英文平台改写
- 帮我把这章改得更跨文化、更有移动端节奏

## 何时使用

- 把任意题材章节改写成更适合 WebNovel 的英文版本。
- 需要强化跨文化可读性、因果链清晰度、移动端节奏与英文平台门禁。
- 需要为题材包装层提供统一的 WebNovel 平台共性骨架。

## 不适用情形

- 只做题材边界补充，不做平台输出本体。
- 只做多平台编排、落盘和终检，而不处理 WebNovel 平台风格本身。

## 本层职责

- 维护 WebNovel 平台的跨题材共性规则。
- 统一承接 WebNovel 版输出的结构约束、英文门禁、标题倾向与风格门禁。
- 为 `题材名-输出WebNovel版` 提供稳定的通用平台骨架。

## 默认执行顺序

1. 先读取对应题材的 `题材名-输出WebNovel版` Skill。
2. 先锁定零新增事实、英文门禁、工作目录、章节层级、零中文/CJK 与标题同步规则。
3. 再执行 WebNovel 的开头抓力、中段推进、跨文化可读性与章末强钩子强化。
4. 最后才处理降相似度，不得为了降重破坏英文可读性与平台节奏。

## 默认输出口径

- 默认输出一版可直接继续落盘或进入平台终检的 WebNovel 英文派生正文。
- 默认保留原章核心信息、因果链清晰度与跨文化可读性，不新增关键事实。

## 最低交付

- 有明确的英文顺滑度、跨文化可读性与章末驱动。
- 保住原章核心事件链、因果链与移动端追读节奏。
- 读起来不像直译稿，也不像失去平台张力的泛英文稿。

## 硬规则

- 命中本技能时，必须同时加载对应题材的 `题材名-输出WebNovel版` Skill。
- 题材特有规则不得回写到本文件中平行维护。
- 修改 WebNovel 平台共性规则时，应优先修改本 Skill，而不是多个题材入口。
- 平台派生正文默认落在本 Skill 的工作目录 `WebNovel/` 下。
- 章节级派生正文目录统一按小说结构决定：若作品有分部，则位于 `WebNovel/第X部/第Y卷/`；若作品无分部，则位于 `WebNovel/第X卷/`。
- 平台派生正文文件名默认不带日期，沿用既有章节号与平台标题同步规则；日期只用于配套审阅报告、书评等派生产物。
- 平台稿落盘后必须显式运行字数门禁脚本：正文至少用 `scripts/platform_validate.ps1` 以 `minLen=6500, requireAfterword=true, minAfterwordWords=100, maxAfterwordWords=150` 校验整稿；若只单独核 `## 作者有话说`，再用 `scripts/count-afterword-words.ps1 -MinWords 100 -MaxWords 150`；除 `## 作者有话说` 标记行外，还必须用 `scripts/check_no_cjk_except_marker.ps1` 校验零中文 / CJK。
- 在无损迁移完成前，不得删除源 `输出WebNovel版.prompt.md` 中尚未完成映射的内容。

## 与其他 Skill / Prompt 的边界

- 本 Skill 只负责 WebNovel 平台的跨题材共性骨架。
- 题材边界、题材禁行项与题材特有口径，继续由对应 `题材名-输出WebNovel版` 承接。
