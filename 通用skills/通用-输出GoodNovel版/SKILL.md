# 通用-输出GoodNovel版

这是平台共性本体，负责承接“GoodNovel 版输出”的跨题材共性规则。

## 常见触发词 / 用户说法速查

- 输出GoodNovel版
- 改成 GoodNovel 能发的版本
- 做 GoodNovel 英文平台改写
- 帮我把这章改得更有情绪黏性和关系张力

## 何时使用

- 把任意题材章节改写成更适合 GoodNovel 的英文版本。
- 需要强化情绪黏性、关系张力、顺滑英文与持续阅读意愿。
- 需要为题材包装层提供统一的 GoodNovel 平台共性骨架。

## 不适用情形

- 只做题材边界补充，不做平台输出本体。
- 只做多平台编排、落盘和终检，而不处理 GoodNovel 平台风格本身。

## 本层职责

- 维护 GoodNovel 平台的跨题材共性规则。
- 统一承接 GoodNovel 版输出的结构约束、英文门禁、关系权重与风格门禁。
- 为 `题材名-输出GoodNovel版` 提供稳定的通用平台骨架。

## 继续读取的 references

- `references/平台共性执行细则.md`
- `references/分节级补救映射与详细规则回填.md`
- `references/平台研究补充_GoodNovel都市悬疑.md`

## 默认执行顺序

1. 先读取对应题材的 `题材名-输出GoodNovel版` Skill。
2. 先锁定零新增事实、英文门禁、路径映射、零中文/CJK 与标题同步规则。
3. 再执行 GoodNovel 的情绪黏性、关系张力与追更动机强化。
4. 最后才处理降相似度，不得为了降重破坏英文顺滑度与平台张力。

## 默认输出口径

- 默认输出一版可直接继续落盘或进入平台终检的 GoodNovel 英文派生正文。
- 默认保留原章核心信息、关系张力与追更驱动，不新增关键事实。

## 最低交付

- 有明确的英文顺滑度、情绪黏性与关系拉力。
- 保住原章核心事件链、角色关系与英文平台可读性。
- 读起来不像直译稿，也不像失去故事张力的降重稿。

## 硬规则

- 命中本技能时，必须同时加载对应题材的 `题材名-输出GoodNovel版` Skill。
- 题材特有规则不得回写到本文件中平行维护。
- 修改 GoodNovel 平台共性规则时，应优先修改本 Skill，而不是多个题材入口。
- 平台派生正文文件名默认不带日期，沿用既有章节号、平台目录与路径映射规则；日期只用于配套审阅报告、书评等派生产物。
- 在无损迁移完成前，不得删除源 `输出GoodNovel版.prompt.md` 中尚未完成映射的内容。

## 与其他 Skill / Prompt 的边界

- 本 Skill 只负责 GoodNovel 平台的跨题材共性骨架。
- 题材边界、题材禁行项与题材特有口径，继续由对应 `题材名-输出GoodNovel版` 承接。
