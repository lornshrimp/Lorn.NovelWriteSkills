# 章节控制卡到场景责任映射表

适用于把章节控制卡真正拆成场景单元施工任务，避免控制卡写得很细，场景落地时又重新散掉。

## 一、先明确：场景单元不是重复控制卡，而是执行控制卡

章节控制卡回答的是：

- 这一章为什么非写不可
- 中段回报是什么
- 章末钩子是什么
- 本章要推进哪条线

场景单元回答的是：

- 这一场具体谁动、怎么动、哪一步出后果
- 这一场承担本章哪一段责任

## 二、控制卡向场景层至少要翻译六项

### 1. 章级身份 → 场景职责序列

如果一章已经确定为若干场景单元，那么每场至少应承担以下之一：

- 抓眼启动
- 推进 / 试探
- 中段回报
- 升级 / 代价
- 章末钩子

不要让每一场都只是“继续推进”。

### 2. 主推进线 → 本场目标

- 控制卡里的主推进线，必须被拆成每场的具体目标。
- 若一场说不清自己在主推进线上负责什么，这一场大概率只是过场。

### 3. 本章主抓手 → 本场抓手使用点

- 章节控制卡中的主抓手，必须落实到某一场被真正使用，而不是只在章级说明里存在。
- 若是显性回报章，主抓手最好在本章前半或中段就显影一次。

### 4. 中段回报 → 指定场景

- 控制卡里的中段回报，必须明确落在哪一个场景单元。
- 该场不只是“发生了信息变化”，还要有动作 / 对话 / 证据 / 后果支撑。

### 5. 风险扫描 → 场景风险分配

- 控制卡里列出的风险，不应由全章平均承担。
- 应明确：
  - 哪一场最易空转
  - 哪一场最易说明书化
  - 哪一场最易把钩子写假

### 6. 章末钩子 → 最后一场结尾动作

- 章末钩子必须落成最后一场里的具体动作、揭露或未完成动作。
- 不能让全章写完后，再在结尾硬补一个抽象问题句。

## 三、控制卡 → 场景的最小映射表

| 控制卡字段 | 场景层要回答什么 | 最好落在哪类场景 |
| --- | --- | --- |
| 核心改变 | 哪一场真正触发这一改变 | 中段回报场 / 升级场 / 钩子场 |
| 主推进线 | 每场各推进哪一步 | 全场都要分担，但不可平均发散 |
| 回归压力 | 哪一场让旧债 / 冷线碰回来 | 前半试探场 / 中段反咬场 |
| 中段回报 | 明确落在哪一场 | 第 2–4 场之间优先 |
| 主要风险 | 哪一场最易写崩 | 逐场指定 |
| 章末钩子 | 哪一场收尾、如何收 | 最后一场 |

## 四、常规 3–5 场切法

### 3 场版

1. 启动场：异常 / 风险 / 情绪缺口入场
2. 回报场：试探、误判、回报兑现
3. 钩子场：升级、代价、章末新问题

### 4 场版

1. 启动场
2. 试探场
3. 回报场
4. 钩子场

### 5 场版

1. 启动场
2. 试探场
3. 中段回报场
4. 升级 / 失手场
5. 章末钩子场

不是每章都必须 5 场，但至少要能说明：哪一场负责回报，哪一场负责钩子。

## 五、显性回报章与保温章的拆法差异

### 显性回报章

- 至少有一场明确承担“回报兑现”
- 主抓手必须真正被调用或被证明失效
- 章末更适合把局面推向新的更难问题

### 保温章

- 可以不做最大回报，但要让供血线不断
- 至少一场负责“续火”：微反常、微证据、关系顶歪、规则试错
- 章末要清楚交代下一次显性回报由谁接棒

## 六、场景层失败警报

出现以下任两项，建议回到控制卡重新分配：

- 每场都能单独概括，但合起来没有章级节拍
- 不知道哪一场负责中段回报
- 不知道最后一场具体靠什么形成钩子
- 主抓手整章都没真正用上
- 本章主推进线被平均摊薄到每场都像流水账

## 七、一句话收口

**章节控制卡解决“这一章为什么写”，场景映射表解决“这一章分别由哪几场把它写出来”。**
