# 删解释保信息的压缩动作卡

适用于这种场景：

- 你已经看出一段写得啰嗦
- 你知道解释太多会拖慢
- 但你又不能随手一删，因为里头有功能句、交代句、规则句、动机句

重点解决：**越修越短，但不能把功能句修没。**

## 一、先判断：这段解释到底在干什么

不要一看见解释就想删。

先分清它属于哪一类：

### 1. 事实交代

- 时间、地点、编号、流程、关系、规则边界

### 2. 动机交代

- 为什么这么做
- 为什么不敢这么做

### 3. 因果交代

- 前面发生了什么，导致现在这样

### 4. 读者防迷路交代

- 如果不说，读者就会断线

真正要压缩的不是“信息”，而是**信息的展开方式**。

## 二、压缩前先圈“功能句”

功能句就是删掉后会出事的句子。

常见功能句：

- 没它就看不懂人物决定
- 没它就接不上因果
- 没它就搞不清程序和代价
- 没它就会把一个误会读成漏洞

### 快速判断法

删掉一句后问：

- 读者还看得懂吗？
- 角色现在的动作还成立吗？
- 这条风险还说得通吗？

如果三问里有一问明显出问题，这句就是功能句。

## 三、四种最常用的压缩法

### 1. 解释并列化

把“因为 / 所以 / 其实 / 原来”一长串解释，改成更硬的并列句。

- 弱：因为他之前已经见过这张回执，所以他知道编号不对。
- 强：他见过这张回执。编号不是这个。

适合：

- 事实明确
- 不需要过多过渡情绪时

### 2. 解释动作化

把一句说明塞回动作或反应里。

- 弱：她知道不能再问下去，因为再问就会暴露自己。
- 强：她话到嘴边又咽了回去。再多问一句，就轮到她露馅。

适合：

- 动机句
- 风险句
- 羞耻 / 恐惧 / 心虚类交代

### 3. 解释物件化

把抽象说明借一个物件、界面、回执、门牌、记录载体落地。

- 弱：这套流程很麻烦，而且一旦出错就会留下记录。
- 强：表单一式三联，错一格，后面两联都得跟着重填。

适合：

- 程序链
- 规则边界
- 技术 / 制度摩擦

### 4. 解释后置化

先保最有力的事件或判断，再把最少量解释压到后一句。

- 弱：他没接电话，因为那个号码他认得，是派出所的。
- 强：他没接。那个号码，他认得。

适合：

- 短打段落
- 贴脸段落
- 需要先保撞击感时

## 四、三种不能乱压的句子

### 1. 规则边界句

如果删短后让规则变模糊，读者就会觉得角色像在随便通关。

### 2. 关键动机句

如果删短后人物只剩动作，没有理由，就会像被作者推着走。

### 3. 关键因果句

如果删短后前后像断电，节奏也许快了，但逻辑死了。

## 五、常见误删现场

### 1. 把必要说明删成“高级留白”

- 作者觉得更克制了
- 读者只觉得莫名其妙

### 2. 把动机删成“神秘感”

- 神秘不等于空白
- 该知道的动机还是要给

### 3. 把程序删成“节奏更快”

- 快了，但世界不可信了

## 六、段级压缩动作

### 1. 一段只保一个主要说明任务

别在同一段里同时讲：

- 背景
- 关系史
- 规则
- 动机
- 旧案

能拆就拆，能延后就延后。

### 2. 重复解释只留最硬的一次

如果前文已经解释过：

- 后文就别换个说法再解释一遍
- 留一次最准的，其他改成人物反应或省掉

### 3. 让说明服务当下，不服务作者焦虑

只保当下这场必须知道的那一点。

不是所有作者知道的，都要此刻告诉读者。

## 七、压缩后的复核动作

压完以后，不要只看“变短没有”。

还要回核：

- 关键事实是否还在
- 关键因果是否还顺
- 人物决定是否还说得通
- 规则代价是否还清楚
- 钩子是否被压没

## 八、一个实用顺序

压缩一段解释时，通用顺序可按：

1. 圈功能句
2. 删重复解释
3. 把能动作化的动作化
4. 把能物件化的物件化
5. 把剩下最必要的说明压成最短句
6. 回核逻辑有没有断

## 九、一句话标准

真正有效的压缩，不是把解释杀光，而是让读者用更少的字，依然完整读懂该懂的东西。
