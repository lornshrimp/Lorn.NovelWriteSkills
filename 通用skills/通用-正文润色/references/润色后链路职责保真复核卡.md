# 润色后链路职责保真复核卡

适用于正文润色完成后，确认文本是不是“更顺了但没虚”，以及是否只是把上游结构问题抹平了表面。

## 一、先明确：润色不是用来掩盖结构失配

有些章节润色后读起来更顺，但实际上发生了这些事：

- 章首抓力被磨成更柔顺的说明
- 中段回报被写得更顺，却更不显眼
- 章末钩子被润成更漂亮的句子，却更不想翻页
- 场景职责边界被润平，导致每场都像一个口气

所以润色后必须做一次链路复核。

## 二、最低复核对象

至少复核以下四项：

1. 本章主卖点有没有被润淡
2. 中段回报有没有被润虚
3. 章末钩子有没有被润圆
4. 对话 / 场景职责有没有被润平均

## 三、三类常见假改善

### 1. 更顺了，但更弱了

- 句子更圆
- 但异常、危险、代价变轻

### 2. 更自然了，但更少事了

- 去掉了一些刺点
- 连同局面变化一起去掉了

### 3. 更漂亮了，但更不像这章该有的职责

- 第一场不再抓眼
- 中段回报不再像回报
- 结尾不再像钩子，而像余韵

## 四、润色后必须重问的五个问题

1. 第一场还清不清楚自己负责抓眼 / 启动？
2. 中段那一场回报还显不显眼？
3. 最后一场还像钩子场，还是被润成收束场？
4. 主抓手是不是还在该显影的位置上？
5. 如果删掉“漂亮句”，这章的结构职责还站得住吗？

## 五、何时只继续润色，何时回写上游

### 可继续停留在润色层

- 职责都还在，只是表达不够准
- 场景边界仍清楚
- 卖点、回报、钩子都仍可指认

### 必须回写上游

出现以下任两项时：

- 第一场被润完后不再抓眼
- 中段回报只能靠解释指出来
- 章末钩子被润平后，下一章接口不清了
- 原本不同职责的场景被润成同一口气
- 卖点只剩主题感，不再有抓手感

回写优先级：

1. 先回写场景职责
2. 再回写章节控制卡
3. 必要时再回开头 / 章末强化层

## 六、最小复核表

| 润色后暴露的问题 | 优先回写哪里 | 典型补法 |
| --- | --- | --- |
| 第一场不再够抓 | 场景职责 / 章首强化 | 恢复抓眼职责与前段异常显影 |
| 中段回报被润淡 | 控制卡 / 场景职责 | 重新钉死回报场与回报载体 |
| 章末不再想翻页 | 章末强化 / 场景职责 | 恢复钩子场职责 |
| 场景边界被润平均 | 场景职责 | 重新拉开各场职能 |
| 卖点被润成抽象感受 | 控制卡 | 恢复主抓手与卖点显影位 |

## 七、一句话收口

**润色成功的标准不是“更顺”，而是“更顺之后，章首、中段、章末和卖点职责都还在，而且更能被读者感到”。**
